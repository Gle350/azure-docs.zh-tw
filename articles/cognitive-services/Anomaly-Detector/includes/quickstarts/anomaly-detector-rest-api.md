在本快速入門中，您將了解如何使用異常偵測器服務和 cURL，偵測時間序列資料批次中的異常狀況。

如需有關異常偵測器概念的高階探討，請參閱[概觀](../../overview.md)文章。

## <a name="prerequisites"></a>必要條件

- Azure 訂用帳戶 - [建立免費帳戶](https://azure.microsoft.com/free/cognitive-services)
- 擁有 Azure 訂用帳戶之後，在 Azure 入口網站中<a href="https://ms.portal.azure.com/#create/Microsoft.CognitiveServicesAnomalyDetector"  title="建立異常偵測器資源"  target="_blank">建立異常偵測器資源<span class="docon docon-navigate-external x-hidden-focus"></span></a>，以取得您的金鑰和端點。 部署完成後，選取 [移至資源] 按鈕。
    - 您需要所建立之資源中的金鑰和端點位址，才能使用 REST API。 
    您可以使用免費定價層 (`F0`) 來試用服務，之後可升級至付費層以用於實際執行環境。

## <a name="detect-anomalies-for-an-entire-series"></a>偵測整個序列的異常情況

在命令提示字元中執行下列命令。 您必須將下列值插入命令中。
- 您的異常偵測器服務訂用帳戶金鑰。
- 您的異常偵測器端點位址。 
- 時間序列資料的有效 JSON 檔案，用來測試異常狀況。 如果您沒有自己的檔案，可以從[要求本文範例](https://westus2.dev.cognitive.microsoft.com/docs/services/AnomalyDetector/operations/post-timeseries-entire-detect)建立 sample.json 檔案。

```curl
curl -v -X POST "https://{endpointresourcename.cognitive.microsoft.com}/anomalydetector/v1.0/timeseries/entire/detect"
-H "Content-Type: application/json"
-H "Ocp-Apim-Subscription-Key: {subscription key}"
-d "@{path_to_file.json}" 
```

如需填入所有值的範例：

```curl
curl -v -X POST "https://my-resource-name.cognitiveservices.azure.com/anomalydetector/v1.0/timeseries/entire/detect" -H  "Content-Type: application/json" -H "Ocp-Apim-Subscription-Key:1111112222222ed333333ab333333333" -d "@test.json"
```
